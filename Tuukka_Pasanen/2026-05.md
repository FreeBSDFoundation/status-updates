One can define a variable from the command line with `spdxtool --define-variable=variable_name=variable_value`, and then the variable is added to the global namespace and included in any `.pc` file.

## **Reporting Period:** 2025-05-06 - 2026-05-12

### Project: SBOMs (CRA)
#### **Overall Status:** On Track
#### **Summary**
All application binary files installed to `/usr`, `/sbin`, `/usr/bin` and `/usr/sbin` in FreeBSD 15.0 have been covered with `.pc`-files and they can be converted to SPDX 3.0.1 SBOM with `pkgconf`. Since `pkgconf` and other tools are part of the FreeBSD base, upstreaming `.pc`-files should begin as soon as possible.

A feature has been added to define variables for upstreaming `pkgconf`'s `spdxtool` and `bomtool`. This allows the use of dynamic values similar to `${FREEBSD_VERSION}` in `.pc` files, eliminating the need for hardcoded versions.

The OSV `pkg(1)` command PR has been updated in preparation for the upcoming 'lunch and learn' webinar. The latest version has been updated to latest git and old and new tests have been converted to new format.
