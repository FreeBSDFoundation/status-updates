# 05 2026 Status Update for Roman Bogorodskiy

## **Reporting Period:** [2026-05-04] - [2026-05-10]

### Project: libvirt on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

Mainly focused on the qemu agent support. Currently implemented two
APIs: one to send arbitrary commands and another one is to retrieve the
guest's hostname. This works, but there are some major items to work on, mainly
figuring out how to share the qemu guest agent related code with the Qemu
driver. I have tried a couple of approaches, but neither I like them nor
they actually work. I plan to polish the remaining bits and send out an PoC/RFC
for ideas from the upstream on this subject.

#### **Completed Tasks**

 * Domain boot timeout patch was merged into libvirt-tck.
 * Documentation updates for the libvirt bhyve driver were merged.

#### **Blockers**

Outline of the `bhyve` issues that are either blockers or create
a noticeable inconvenience for development:

 * [bootindex not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=279720),
 * [resume issues](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=290735),
 * [tpm issues](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=287326),
 * [virtio-input not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=290364),
 * [arm64 CPU frequency reporting not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=291810),
 * Lack of capabilities reporting in bhyve.

#### **Next Steps**

 * Polish CI runs against the snapshot.
 * Continue the qemu-guest-agent work.
 * Look into extending `rctl(8)` support. Specifically, memory and swap hardlimits.
