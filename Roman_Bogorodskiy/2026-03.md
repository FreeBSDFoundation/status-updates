# 03 2026 Status Update for Roman Bogorodskiy

## **Reporting Period:** [2026-03-30] - [2026-04-05]

### Project: libvirt on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

Completed initial implementation of the `virtio-console` support
for the libvirt bhyve driver and published the
[patch](https://lists.libvirt.org/archives/list/devel@lists.libvirt.org/thread/5MT37BGWT4A5K56QP3DOEUYPCUQN4PZE/)
for review.

I also implemented a draft version of the `virDomainQemuAgentCommand` API
(or `virsh qemu-agent-command`), which allows running arbitrary Qemu Guest Agent commands.

Also started experimenting with resource usage limitation using `rctl(8)`.
This proof of concept starts with I/O operations throttling.
This is modeled slightly differently in libvirt compared to `rctl(8)` as libvirt
works in per device scope, while `rctl(8)` works in per process scope.
Still need to figure out what is the best way to approach that.

#### **Completed Tasks**

 * Ports updated to the latest release, 12.2.0.

#### **Blockers**

Outline of the `bhyve` issues that are either blockers or create
a noticeable inconvenience for development:

 * [bootindex not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=279720),
 * [resume issues](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=290735),
 * [tpm issues](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=287326),
 * [virtio-input not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=290364),
 * [arm64 CPU frequency reporting not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=291810),
 * Lack of capabilities reporting in bhyve.

#### **Next Steps**

 * Finish the CI effort for testing against VM images.
 * Continue working on the qemu-guest-agent integration.
 * Continue researching `rctl(8)` integration.
