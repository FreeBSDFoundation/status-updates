# 07 2026 Status Update for Roman Bogorodskiy

## **Reporting Period:** [2026-07-13] - [2026-07-19]

### Project: libvirt on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

This week I have implemented a TCK test for the domain rename API which
I have implemented earlier. The test is now
[merged](https://gitlab.com/libvirt/libvirt-tck/-/merge_requests/83).

Implemented the `virDomainGetGuestInfo()` for the libvirt bhyve driver.
That is one more API on top of the qemu guest agent which allows to query
various general information about the guest such as OS type and
version, users, loadavg, etc. This is currently under
[review](https://lists.libvirt.org/archives/list/devel@lists.libvirt.org/thread/KOS4YQQBP4TP3F6QX5L5S7HJSRBH7SCW/).

Implemented the guest disk statistics reporting using the qemu guest agent's
`guest-get-diskstats` command. The agent does not implement this command
on FreeBSD, I will look into implementing it as in my TCK-based CI I use only FreeBSD
guests and it would be nice to be able to test it there.

My CI is now running TCK tests against the FreeBSD -CURRENT snapshots on a weekly basis.
I had also uploaded the
[script](https://github.com/novel/libvirt-freebsd-ci/blob/master/bin/test_freebsd_snapshot.sh)
that does most of the job. I still need to extend the Jenkins job for a better error reporting,
e.g. mask some tests that are known to fail in this environment.

#### **Completed Tasks**

 * `virDomainRename()` TCK tests are merged.
 * Domain startup error handling is merged.

#### **Blockers**

Outline of the `bhyve` issues that are either blockers or create
a noticeable inconvenience for development:

 * [bootindex not working](https://bugs.freebsd.org/bugzilla/show_bug.cgi?id=279720),

 * Lack of capabilities reporting in bhyve.

#### **Next Steps**

 * Continue implementing APIs on top of the qemu guest agent.
