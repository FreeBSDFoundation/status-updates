## **Reporting Period:** 2026-02-25 - 2026-03-03

### Project: Improve OpenJDK on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

This week, OpenJDK 25 was updated to version 25.0.2, and there was also some changes to the headless flavor of the port. It appears that the full Xorg dependencies are needed also for headless builds. This surfaces as we the ports no longer depend on previous ports that install these dependencies.

A baseline branch has been established in the [openjdk/bsd-port](https://github.com/openjdk/bsd-port) repository on Github. This will serve as a living blueprint of what eventually should become the upstream BSD port. The baseline is a clone of the [existing out-of-tree port](https://github.com/openjdk/bsd-port/), and will be kept up to date with it.

The [first PR](https://github.com/openjdk/bsd-port/pull/3) to the actual upstream BSD port branch has also been made, in part to test the process and workflow, but ultimately a patch we want to land. The upstream community seems to have received it well, and are helpful with guiding us in this new phase of the project.

Meanwhile work on keeping the existing out-of-tree BSD port up to date continues, and is getting close to complete.

#### **Open Questions and Discussion**

Got some feedback from one OpenJDK user that it would be nice to include the upstream release notes in whenever we update the ports to the latest upstream releases. It's not always obvious from the version numbers if the release contains potentially breaking changes. The case prompting this was the release of OpenJDK 11.0.30, which disabled all `TLS_RSA` cipher suites, which causes some disruptions in their internal network.

There's concern that adding more work to the release process will strain the already time constrained task too much. We're considering a low effort approach that will not burden the maintainers with extra work once in place.
