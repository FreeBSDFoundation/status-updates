## **Reporting Period:** 2026-03-11 - 2026-03-17

### Project: Improve OpenJDK on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

The past week was mostly spent on getting up to date and starting the port for OpenJDK 26. Apart from backporting some of the fixes we have done lately for mailnline, the port itself looks like it's going to be fairly straight forward. Also started on updating and backporting BSD fixes for OpenJDK 25. Since this is an LTS release, it is being kept updated beyond the normal two patch releases normally done by the OpenJDK project.

While the PR for the upstream BSD port has received an accepting review from an upstream member, it's still not ready to be integrated, as we also need a review from a member with a role in the BSD port project. Which essentially means Kurt, Greg or me.

#### **Challenges**

Getting reviews from upstream project members on the patches to the [bsd-port repo](https://github.com/openjdk/bsd-port) is required to make sure the patches will later be accepted into the mainline repo when the time comes. However, attracting the right people from the different parts of the JDK has to be done manually for now.

For patches directly to the mainline repo, the Skara bot adds labels that ensures that the discussion in the pull requests are mirrored to the correct mailing lists. However this is not done for project repos, like the bes-port repo. We are currently investigating the best way to get around this, whether we have to manually entice people to review every change, or if we could [get some help](https://bugs.openjdk.org/projects/SKARA/issues/SKARA-2679?filter=allopenissues) from the Skara bot.

## **Reporting Period:** 2026-03-04 - 2026-03-10

### Project: Improve OpenJDK on FreeBSD
#### **Overall Status:** On Track
#### **Summary**

Landed a patch that enable building the Hotspot Debugger (HSDB) for FreeBSD, when using the LLVM backend. This was needed by one of the tests for OpenJDK on Aarch64 that checks whether the correct sequence of opcodes are generated for some calls. Also finally nailed a long standing issue where a test would fail often, but not all the time due to the ordering of loaded shared objects in the process being debugged.

The attempt at merging the first PR to what will become our upstreaming branch is still ongoing.
