### Project: CPPC / Power Management
#### **Overall Status:** On Track

[D55474](https://reviews.freebsd.org/D55474) - acpi_cppc: Add cppc library for parsing ACPI _CPC object (new diff uploaded, needs revision)
Uploaded new diffs addressing olce@'s comments (turn KASSERTs into error returns, drop SYS_RES_FFH while keeping acpi_ffh_read/write). Still needs revision: read-only items should keep the full register specification since some are dynamic at runtime, and register access should use the GAS BitWidth instead of hardcoding bus_read_8().

[D58188](https://reviews.freebsd.org/D58188) - x86: Support ACPI CPPC FFH (new)

[D57825](https://reviews.freebsd.org/D57825) - hwpstate_amd: Detect ACPI _CPC object to enable CPPC (new diff uploaded)
Updated the patch per olce@'s suggestion to guard the ACPI check with the HWPFL_USE_CPPC flag so acpi_cppc_ctx_init() is skipped when CPUID says CPPC is unsupported. The patch itself looks good to the reviewer; the remaining concern is the dependency on the CPPC library ([D55474](https://reviews.freebsd.org/D55474)), with a suggestion to break it by directly checking whether _CPC exists in the ACPI namespace.
