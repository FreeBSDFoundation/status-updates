  `process plugin pcpu pc_curthread` then play with the value to do other stuff (e.g. check values in `pc_curthread`).

### Side Project: EEVDF Scheduler on FreeBSD
#### **Overall Status:** On Track
#### **Summary**
FreeBSD's two schedulers, 4BSD and ULE, are based on interactive scores. Since it is nearly impossible to have scheduler that
performs ideally in every environment, introducing fairness-based (ULE is still fair, but) scheduler for FreeBSD not only gives
wider range of options for users but also ensures that our kernel code isn't dependent on a single scheduler.

This has been abandoned since February as the red-black tree review didn't get any attention. As FreeBSD's existing WAVL
implementation is unusable for this kind of use (see below), the new RB tree implementation is in the `sched_eevdf.c` file.

The draft is available at [my Github fork](https://github.com/mchoo7/freebsd-src/tree/eevdf), but this only compiles will cause
kernel panic. **SO DO NOT RUN THIS ON ANY MACHINE.**
#### **Completed Tasks**
- Gave a shot to Claude Opus 4.6 to write EEVDF implementation for FreeBSD. Claude Desktop achieved <30% of what I wanted
  (the code didn't even compile) and Claude Code did ~70% (code compiled at least). But I'm not satisfied with the result yet
  and expect that the scheduler will cause kernel panic when I try to boot a machine with it.
#### **Features Delivered**
#### **Challenges**
- The code is LLM-generated, so it's quite unreadable. I need to rewrite some functions and comments to match `sched_ule.c`'s style.
- Scheduler is essential to any operating systems, so I cannot trust LLM like I did for other jobs (e.g. shell scripting, CSS styles).
  It'll take some time to audit that ~3k LoC and compare it with another ~4k LoC from `sched_ule.c`.
#### **Blockers**
#### **Next Steps**
#### **Open Questions and Discussion**
- For now, I copy-pasted nice-priority table from Linux (~10% exponential curve). We need numerous benchmarks to find best values
  for FreeBSD. This exponential curve table is something I wish ULE to have (olce@ said he's working on this).
- Because this started as my side project to see how well LLM generates code, I cannot guarantee its code quality before auditing
  is done. If other developers refuses to land this even after successful results (due to policies, copyrights, personal belief, etc),
  I'm fine with abandoning it.
