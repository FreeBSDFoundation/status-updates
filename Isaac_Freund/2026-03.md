## **Reporting Period:** 2026-03-02 - 2026-03-10

I continued working on the pkg bug I found last week, with the goal of finally
finishing the pkgbasify robustness improvements.

#### **Work in progress**

- I continued work on a fix for the pkg bug where `install --register-only`
  followed by `install -f` overwrites config files without merging or creating
  a .pkgsave/.pkgnew backup.
    - `install -f` is handled differently from upgrades internally and never tries to merge
    - Changing `install -f` to merge configs is not sufficient however, since the config file contents in the pkg database are empty.
    - Seems that the config file contents in the database may not be intentional, but rather a separate but related bug with pkg's config file handling
