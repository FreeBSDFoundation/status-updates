  - [D57412](https://reviews.freebsd.org/D57412) ("acpi: Fix panic when
    reading 'hw.acpi.suspend_state' knob on S1/S2")
  - [D57413](https://reviews.freebsd.org/D57413) ("acpi: Move supported states
    arrays into the softc")
  - [D57414](https://reviews.freebsd.org/D57414) ("acpi: Forbid requesting S4
    when S4BIOS is not supported")

    repo](https://github.com/OlCe2/freebsd-src/compare/main...ffe6e2cbd02a260c5f).
- From the rebased `oc-hibernate` branch, isolate changes about ELF headers
  (type, content) into two separate commits, shareable with `kib@`'s loader
  branch (`hiber`), and do the impacts.
- Fix bugs in and around triggering image saving (and more bugs in `acpi.c`),
  including wrong reported size of program headers, not freeing trampoline memory
  (allocated for the loader) in case of suspend bounce, etc.
- Work on an early prototype of resume in the kernel, once the loader has handed
  off control, in order to check see what is involved on the kernel side and
  that the interface with the loader is viable.  Make slight amendments to that
  interface as discussed last week with `kib@`.
- Propose amendments to the `hiber` branch: Integrate there the two same commits
  from `oc-hibernate`, and amend the existing code to use the new definitions.
- Generate a new hibernate dump image.  Extract the ELF file from it and check
  that the ELF headers and the program headers have the expected content.
- Send to `kib@` an email explaining where to download the new ELF image, and
  where to find the current `oc-hibernate` and updated `hiber` branches.
- Start working on BSDCan 2026's presentation.
