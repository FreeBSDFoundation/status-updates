### Dynamic P-state support (CPPC) \[Laptop Project\]

Long discussion with ShengYi (`aokblast@`).  We are going to have a single EPP
global control (per driver at first) that affects all cores.  For AMD, CPPC
support will by default be disabled if the `_CPC` ACPI table is not present,
even if CPUID indicates it is (I'm still unsure if this is the best move).
I suggested a knob to control that, being a mask of non-present "features" to
ignore when deciding to enable CPPC (bit 0 would represent the absence of
`_CPC`, bit 1 would represent the absence of the CPU flag).  We also discussed
about a global profile knob, with predefined values, and one special value
"custom", which is not settable but reported when individual knobs have been
modified separately (alternative proposed by ShengYi: force the administrator to
set it to "custom" before it is allowed to modify individual knobs; I do not
find it convenient as it is quite unique and surprising to users).  I will
implement setting/getting the minimum, maximum and desired performance knobs in
the Intel driver, after some refactoring/improvements.
