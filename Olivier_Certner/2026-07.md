## **Reporting Period:** 2026-07-08 - 2026-07-14

### CPPC

- Review [D57825](https://reviews.freebsd.org/D57825) ("hwpstate_amd: Detect
  ACPI _CPC object to enable CPPC"): Some suggestion.  Looks OK modulo the
  dependency on the "CPPC library" (see next revisions) which, in the interest
  of getting this in the tree earlier, could be replaced by a simple test on the
  ACPI `_CPC` table's presence.
- Review [D55476](https://reviews.freebsd.org/D55476) ("x86: Implement
  Functional Fixed Hardware in SYS_RES"): Reject: FFH (ACPI Fixed Function
  Hardware) cannot be a resource type like memory or I/O space, as a such an
  "address" meaning depends on the attribute where it is used.  FFH is only
  mapped to MSRs in certain circumstances.
- Review [D55475](https://reviews.freebsd.org/D55475) ("acpi: Support Functional
  Fixed Hardware"): Reject for the same reasons (note the changes there that
  show even more clearly that this is wrong conceptually).
- Review [D55474](https://reviews.freebsd.org/D55474) ("acpi_cppc: Add cppc
  library for parsing ACPI _CPC object."): The rejection of previous revisions
  has several impacts here.  Additionally, ask for specific changes.
- Review [D55505](https://reviews.freebsd.org/D55505) ("acpi_perf: Mark readonly
  when using FFH"): Reject, this makes no sense since FFH cannot be a resource
  (no `SYS_RES_FFH`).
- Commit [1a3d61552546](https://cgit.freebsd.org/src/commit/?id=1a3d61552546)
  ("hwpstate_amd(4): Restore CPU ID cache after introducing cpu_get_pcpuid()").
