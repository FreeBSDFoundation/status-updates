## **Reporting Period:** 2026-03-18 - 2026-03-24

### Project: KDE installer dialogs
#### **Overall Status:** On Track for 15.1
#### **Summary**

The idea is: a user installs KDE at the end of bsdinstall through a series of dialog-based interfaces.
After rebooting, sddm starts and the user can log into the graphical environment.
[Repository](https://gitlab.com/alfix/kde-installer-dialogs).

**Weekly Work Report:**

- Created ([desktop.bsdinstall](https://gitlab.com/alfix/kde-installer-dialogs/-/blob/main/desktop.bsdinstall)) a script version to be integrated in bsdinstall.
   - Tested by including it in locally generated ISOs based on CURRENT 16 and STABLE 15.
   - Successfully installs GPU drivers and KDE during a BSDInstall session.
   - Improved GPU autodetection and driver selection menus. Current test failures are mostly due to incorrect user GPU selection; this step is still a work in progress.

#### **Completed Tasks**

 - The script installs and set up: gpu driver, xorg, KDE Plasma, and sddm after a fresh installation.
 - The script integrated in a generated ISO installs the GUI environment via bsdinstall.

#### **Features Delivered**

 - The _desktop_ script, tested with: VESA, SCFB, Intel, and AMD.
 - Added menu to select NVIDIA GPUs (tested via Call for Testing).
 - GPU auto-detection.
 - VirtualBox option in GPU menu, auto-detection, and pkgs installation.
 - VmWare option in GPU menu, auto-detection, and pkgs installation.
 - Add Option to install extra NVIDIA GPUs tools (tested via forum).
 - The _desktop.bsdinstall_ script, to install GPU drivers and KDE during a BSDInstall session.

#### **Challenges**

 - Testing GPU drivers, mainly NVIDIA.

#### **Next Steps**

 - Fix the Ly Display Manager.
 - Review for CURRENT.
