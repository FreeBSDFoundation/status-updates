## **Reporting Period:** [2026-06-22] - [2026-06-28]

### Project: Full CPUID Control for bhyve
#### **Overall Status:** On Track
#### **Summary**
Worked on symbolic CPUID feature names. These names will be translated by the cpuid code into a combination of function, index, register, and bit. Thus it's
possible to enable or disable a feature like this:

```cpuid.enable=fpu,vmx```

Pre-defined CPU model configurations will use this to be more readable. The underlying mechanism (CPUID register/bit manipulation, CPUID constraints) remain the same and can still be used directly if so desired.

#### **Next Steps**
- try to optimize the processing of symbolic CPUID feature names (as, don't set each bit individually, but group them internally by function, index, and register)
- create configurations for CPU types and families
- improved CPUID feature clearing
- CPUID constraint override
- hypervisor signature override
